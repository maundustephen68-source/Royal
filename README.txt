# Royal Furniture Website — Setup Instructions (v2, with secure backend)

## What changed from before

Previously this site was fully static: product prices lived in the HTML and
the cart lived in the browser's localStorage. Anyone could open dev tools,
edit a price, and "check out" at any amount.

Now the site talks to a real backend API (see the separate `royal-backend`
folder) that is the single source of truth for products, prices, and stock.
The browser only ever sends a product ID and quantity — the server looks up
the real price and computes the total. Checkout still ends in a WhatsApp
message (as you wanted), but that message is now generated by the server
from a validated order, not typed by JavaScript in the browser.

## Files in this package

```
royal-furnitures/
│
├── Home.html          ← Home page
├── PRODUCTS.HTML       ← Products page (now loads products from the API)
├── About Us.html       ← About page
├── contact.html        ← Contact page (form now submits to the API)
├── royal.css           ← All styles
├── royal.js             ← All JavaScript — set API_BASE near the top to your backend URL
└── images/              ← Your existing images folder (copy from GitHub) — used on
                            About/Home pages; product images now come from the database
```

## Setup steps

1. Deploy the backend first — see `royal-backend/README.md` for full
   step-by-step instructions (free Postgres via Neon + free hosting via Render).
2. Once deployed, open `royal.js` and set:
   ```js
   const API_BASE = 'https://your-actual-backend-url.onrender.com';
   ```
3. Upload these frontend files (and your `images/` folder, still used for
   the About/Home pages) to GitHub Pages or wherever you host the site, same
   as before.
4. On the backend host, set `ALLOWED_ORIGINS` to your live site's URL so the
   API only accepts requests from your real domain.

## WhatsApp number

Your WhatsApp number **+254 716 817 495** is wired throughout the site and
is also set as `WHATSAPP_NUMBER` in the backend's environment variables —
update it in both places if it ever changes.

## Adding more products

Products now live in the backend's database, not in HTML. See
`royal-backend/README.md` → "Managing products going forward" for how to
add, edit, or remove products via the admin API. The catalog currently
ships with 24 seeded products (12 original + 12 new placeholders across
Living Room, Bedroom, and Office) — replace the placeholder image URLs with
your real product photos whenever you're ready.

## Features

- Mobile hamburger menu
- Active nav link highlighting
- Scroll-reveal animations
- Shopping cart with localStorage (for browsing convenience — always
  re-validated against the server before checkout)
- Product modal with quantity selector
- Server-validated WhatsApp checkout (cart contents & prices verified
  against the database before any WhatsApp message is generated)
- Category filter + search bar on Products page (client-side, over
  API-loaded data)
- Contact form — now actually saved via the backend, with validation and a
  character counter
- Back to top button
- Toast notifications
- Pulsing WhatsApp floating button
